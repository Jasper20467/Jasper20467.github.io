## Imperative Code 和 Declarative Code 的說明與差異

### 1. 定義
- **Imperative Code（命令式編程）：**  
  使用一系列明確的指令來告訴程式如何完成任務，關注「如何做」。開發者需要詳細描述每一步操作的過程。

- **Declarative Code（聲明式編程）：**  
  描述程式的目標狀態，而非具體的操作步驟，關注「做什麼」。開發者只需定義結果，執行細節由底層系統處理。

### 2. 差異比較
| 特性               | Imperative Code                     | Declarative Code                  |
|--------------------|--------------------------------------|-----------------------------------|
| **關注點**         | 如何完成任務（步驟）                | 目標狀態（結果）                  |
| **可讀性**         | 低（需理解每一步操作）              | 高（專注於結果）                  |
| **靈活性**         | 高（可控制每個細節）                | 低（依賴底層實現）                |
| **適用場景**       | 複雜邏輯或需要精確控制的場景        | 配置、聲明需求或簡化操作的場景    |

### 3. 例子
- **Imperative Code 範例（命令式）：**
  ```python
  # 計算 1 到 5 的總和
  total = 0
  for i in range(1, 6):
      total += i
  print(total)
  ```

- **Declarative Code 範例（聲明式）：**
  ```python
  # 計算 1 到 5 的總和
  print(sum(range(1, 6)))
  ```

### 4. 常見應用
- **Imperative Code:** 傳統程式語言（如 C、Java）中的大部分邏輯。
- **Declarative Code:** SQL 查詢、HTML 標籤、基於 YAML/JSON 的配置文件。
